# Meeting Cadence

## Revision History
- 2026-03-04: Initial operating cadence for execution and decision governance.

## Weekly Rhythm
1. Daily async standup:
   - completed
   - in progress
   - blockers
   - next 1-3 actions
2. Twice-weekly engineering review:
   - high-risk changes
   - experiment health
   - test and reliability trends
3. Weekly decision review:
   - go/no-go posture
   - gate criteria status
   - next-cycle priorities

## Required Inputs for Weekly Decision Review
1. `docs/engineering/STATUS_DASHBOARD.md`
2. `docs/engineering/EXECUTIVE_OVERVIEW.md`
3. latest reliability artifacts and key visuals

## Output Expectations
1. Updated decision log entry or explicit "no change".
2. Updated priority list for next cycle.
3. Named owners for critical follow-up tasks.
